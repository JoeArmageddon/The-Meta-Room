Step 1: Prerequisites

Before you begin, ensure you have these accounts and tools

Required Accounts: You'll need to create free accounts on these platforms

1.1 Required Accounts

  • GitHub Account - For repository access
    Sign up at github.com
  • Supabase Account - For PostgreSQL database
    Sign up at supabase.com
  • Groq Account - For AI explanations
    Sign up at console.groq.com
  • Vercel Account (Optional) - For deployment
    Sign up at vercel.com

1.2 Required Software

  • Node.js 18+ - Download from nodejs.org
  • Git - Download from git-scm.com
  • VS Code (Recommended) - Download from code.visualstudio.com
Verify Node.js: Open terminal and run node --version - should show v18.0.0 or higher
🗄️

Step 2: Database Setup (Supabase)

Create and configure your PostgreSQL database

1

Create Supabase Project

Go to supabase.com and click "New Project"

  • Choose your organization
  • Name your project: the-meta-room
  • Set a secure database password (save this!)
  • Choose a region close to you
2

Get Your API Credentials

Once the project is created, go to Project Settings → API

  • Copy Project URL (looks like: https://xxxxx.supabase.co)
  • Copy anon public API key
3

Run Database Schema

In Supabase Dashboard, go to SQL Editor → New Query

  1. Copy the entire contents of lib/db/schema.sql
  2. Paste into the SQL Editor
  3. Click "Run" to execute
✓ Success! Your database should now have all tables created (sources, entries, ai_explanations, relationships, etc.)
⚙️

Step 3: Environment Configuration

Configure your API keys and environment variables

1

Get Groq API Key

Go to console.groq.com/keys

  • Create a new API key
  • Copy the key (starts with gsk_)
2

Create Environment File

In your project root folder, create a file named .env.local

3

Add Environment Variables

Copy this template and fill in your values:

.env.local 
# Supabase Configuration
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...

# Groq API Configuration
GROQ_API_KEY=gsk_your_api_key_here
⚠️ Important: Never commit .env.local to GitHub! It contains sensitive API keys.
NEXT_PUBLIC_SUPABASE_URL
Your Supabase project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
Your Supabase anon/public API key
GROQ_API_KEY
Your Groq API key for AI explanations
📦

Step 4: Installation

Install dependencies and prepare the application

Open Terminal in VS Code or your preferred terminal, navigate to the project folder
1

Navigate to Project Folder

Terminal 
cd D:\Projects_V2\the-meta-room
2

Install Dependencies

This will download all required packages (may take 1-2 minutes)

Terminal 
npm install
3

Verify TypeScript (Optional)

Check for any type errors

Terminal 
npx tsc --noEmit

Should show no errors if setup correctly

✓ Ready! Your application is now installed and ready to launch
🚀

Step 5: Launch the Application

Start the development server

Option A: Development Mode (Recommended for testing)

Terminal 
npm run dev
  • Server starts at http://localhost:3000
  • Auto-reloads when you make changes
  • Best for development and testing

Option B: Custom Port

Terminal 
npm run dev -- --port 3009

Option C: Production Build

Terminal 
npm run build

Then start the production server:

Terminal 
npm start

🎉 You're Ready!

Open your browser and visit http://localhost:3000 to see The Meta Room in action

Troubleshooting

Common issues and solutions

Build Errors

Error: "Missing Supabase environment variables"

Solution: Make sure your .env.local file exists and contains all required variables. Restart the server after adding them.
Error: "Cannot find module"

Solution: Run npm install to ensure all dependencies are installed.

Database Errors

Error: "relation does not exist"

Solution: The database tables haven't been created. Run the SQL schema in Supabase SQL Editor (see Step 2).

Port Already in Use

Error: "Port 3000 is already in use"

Solution: Use a different port: npm run dev -- --port 3009

AI Explanations Not Working

Issue: No AI explanations are generated

Solution:
  1. Verify GROQ_API_KEY is set in .env.local
  2. Check if you have credits in your Groq account
  3. Restart the development server
🌐

Deploy to Vercel (Optional)

Make your app accessible online

  1. Push your code to GitHub
  2. Go to vercel.com
  3. Click "New Project" and import your GitHub repo
  4. Add your environment variables in Vercel settings
  5. Click Deploy!
Tip: For production deployment, consider setting up Row Level Security (RLS) policies in Supabase for better security.
